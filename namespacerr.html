◆ CudaType

enum rr::CudaType
strong

This class contains different types of memory.

This memory can be managed in three different ways: host, device or unified. Host or CPU memory is the one that is usually used by the computer when allocating data and can only be accessed by the CPU. Device or GPU memory is the memory used by the graphics processing unit of your device and can only be accesed by the GPU. Unified memory is a combination of both host and device memory. In this case, both the CPU and the GPU can access the memory. This memory requires additional instructions for memory management.

Enumerator
kHost 

Host memory.

Also known as CPU memory.

kDevice 

Device memory.

Also known as GPU memory.

kUnified 

Unified memory.

A combination of both Host and GPU memory.

kExternal 

Externally allocated memory, not allocated by CudaIsp.
